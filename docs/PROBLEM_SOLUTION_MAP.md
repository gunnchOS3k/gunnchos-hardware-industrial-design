# Problem → Solution Map — gunnchos Hardware Industrial Design

## The human problem
Learners need accessible devices for edge AI and 6G labs without enterprise-only hardware.

Who is harmed: Schools and labs if hardware is unsafe, unaffordable, or non-repairable.

## The technical problem
EVT-0 concepts must mature to manufacturable CAD, electrical design, BOM, and compliance plans.

## The research gap
Existing tools rarely combine **equity**, **open reproducibility**, and **cross-repo evidence** for under-connected communities at Gary-scale fidelity.

## This repo's solution
Device spec packages, diagrams, BOM validation Makefile, affordability/repairability docs.

## What runs today
`make e2e` → `validation logs under results/`

## What the output means
Smoke-test artifact for CI and portfolio review.

## What the output does NOT prove
FCC/CE certification or DVT/PVT manufacturing sign-off.

## How a researcher can extend it
Pick one `next_evidence` item; document benchmark or field protocol; link PR to `[Evidence TODO]` issue.

## How a WAIKE learner can contribute
Run smoke test · file reproduction issue · fix docs/tests · pair with mentor on one evidence issue.
